Git • Commit

¿Qué es un commit y por qué es importante?

Un commit es una instantánea del estado del proyecto guardada en el historial de control de versiones (por ejemplo, Git). Registra los cambios realizados, quién los hizo y cuándo, junto con un mensaje descriptivo.

Por qué es importante

  • Preserva el historial: permite volver a versiones anteriores y entender la evolución del código.
  • Facilita la colaboración: otros pueden ver, revisar y combinar cambios.
  • Aísla cambios: commits pequeños y atómicos hacen más fácil detectar y corregir errores.
  • Soporta buenas prácticas: revisiones de código, integración continua y trazabilidad de cambios.

Buenas prácticas breves

  • Hacer commits atómicos (una sola idea/funcionalidad).
  • Mensajes claros y descriptivos.
  • Ejecutar pruebas antes de commitear.
  • Staging selectivo (`git add` solo lo necesario).
Ejemplos de comandos:
git add archivo.js
git commit -m "Añade función sumar y entrada por consola"
git commit --amend -m "Corrige mensaje del commit"
git revert <hash_del_commit>
